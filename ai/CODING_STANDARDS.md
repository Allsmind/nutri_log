# Padrões de Código

Toda regra de negócio deve ficar em UseCases.

Nunca acessar Repository diretamente pela UI.

Nunca acessar Database pela Presentation.

Todo estado deve utilizar StateFlow.

Todo acesso ao banco deve ser assíncrono.

Toda feature deve conter

Presentation

Domain

Data

Navigation

Utilizar data class sempre que possível.

Evitar Singleton.

Seguir SOLID.

Seguir Clean Code.

Criar testes para regras de negócio.

Comentários apenas quando extremamente necessários.

Priorizar nomes autoexplicativos.
